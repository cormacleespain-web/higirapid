# Best Practice References — Design Lead

Authoritative references for UX, interaction design, and experience architecture.

## UX & Usability

- **Nielsen Norman Group (nngroup.com)**: Usability heuristics, user research, UX guidelines. Use for interaction patterns and evaluation criteria.  
  https://www.nngroup.com/
- **Don’t Make Me Think** (Steve Krug): Clarity and cognitive load. Use when simplifying flows and copy.  
  https://sensible.com/dont-make-me-think/
- **Design of Everyday Things** (Don Norman): Affordances, feedback, mental models. Use for interaction design rationale.  
  (Book; reference for principles.)

## Information Architecture

- **IA Institute**: Information architecture patterns, navigation, taxonomy.  
  https://www.iainstitute.org/
- **Card sorting and tree testing**: Use when defining or validating IA (document method in design docs if used).

## User Research & Journeys

- **User journey mapping**: Document stages, touchpoints, pain points, and opportunities. Use journey template in [templates.md](templates.md). Reference NN/g or UX collective articles on journey mapping for structure.
- **Jobs to Be Done (JTBD)**: Frame user needs as jobs; use when aligning with Product Owner on problem statements.

## Interaction Patterns

- **Material Design (material.io)** and **Apple HIG (Human Interface Guidelines)**: Platform-specific patterns. Use when designing for web, Android, or iOS to align with platform conventions.  
  https://m3.material.io/  
  https://developer.apple.com/design/human-interface-guidelines/

When in doubt, favour clarity and consistency with the product’s design system and accessibility standards over copying a single external pattern.
