# Coordination Protocols

Reference for handoff format and status reporting. These align with `.cursor/rules/team-coordination.mdc`.

## Handoff Document

- **Location**: `docs/_status/handoffs/`
- **Naming**: `[YYYYMMDD]-[HHMM]-[from-agent]-to-[to-agent].md`
- **Sections**: Context, Decisions, Deliverables, Open Questions, Suggested Next Steps

Agents write handoffs when their work is consumed by another agent. The orchestrator uses handoffs to build the next task prompt and to update progress.

## Progress Dashboard

- **File**: `docs/_status/progress.md`
- **Owner**: Team Orchestrator only. Agents do not edit this file.
- **Content**: Current workflow, current phase, completed agents, in-flight agents, blockers, next dispatch(es).

## Decision Log

- **File**: `docs/_status/decisions.md`
- **Format**: Append-only. Each entry: date, agent, decision, rationale, impact.
- **Owner**: Any agent may append when they make a significant decision; orchestrator may consolidate or clarify.

## Project Brief

- **File**: `docs/_status/project-brief.md`
- **Owner**: Team Orchestrator. Updated at start of work and when goal/phase/constraints change.
- **Content**: Product goal, current phase, active feature or initiative, key constraints (time, scope, quality, dependencies).
